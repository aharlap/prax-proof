# Read API

Proof's read API returns activity results for humans, scripts, and AI tools. It
uses separate read keys from the ingest keys described in [docs/embed.md](embed.md).

## Auth

Mint a read key from the Keys page or with:

```sh
curl -X POST /admin/keys \
  -u admin:ADMIN_PASSWORD \
  -H "Content-Type: application/json" \
  -d '{"label":"report reader","kind":"read"}'
```

Read endpoints accept either header form:

```txt
Authorization: Bearer KEY_ID:KEY_SECRET
Authorization: Basic base64(KEY_ID:KEY_SECRET)
```

Admin Basic auth also works:

```txt
Authorization: Basic base64(admin:ADMIN_PASSWORD)
```

Ingest keys cannot read results. Read keys cannot write statements.

## Endpoints

`GET /api/activities`

Lists known activities.

```json
[
  {
    "iri": "https://YOUR-INSTANCE/a/fractions-quiz",
    "name": "Fractions Quiz",
    "pageUrl": null,
    "attempts": 12,
    "completions": 9,
    "lastActivity": "2026-07-03T14:05:01Z"
  }
]
```

`GET /api/activity?slug=fractions-quiz`

activity IRI is not a Proof slug.

Returns a paste-ready markdown report for humans and LLMs.


## Errors

Auth failures return a bare `401` body: `{ "error": "Unauthorized" }`. Other 4xx responses use:

```json
{
  "error": "Activity not found.",
  "docs": "https://github.com/Praxity/prax-proof#api"
}
```

Missing `iri`/`slug` returns `400`. Unknown activities return `404`.

## Caching

Every `/api/*` response sets `Cache-Control: no-store`.
